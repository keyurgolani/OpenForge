# Phase 6 Onboarding Flow

## Current Entry Sequence

1. `/onboarding`
2. authenticated shell
3. workspace overview
4. top-level navigation into Knowledge, Chat, Profiles, Workflows, Missions, Runs, or Artifacts

## Architectural Teaching Goals

The onboarding flow should teach these distinctions early:

- workspaces contain content and execution context
- settings configure infrastructure
- profiles are reusable workers
- workflows and missions are distinct from profiles
- runs and artifacts represent execution and outputs

## Current Gaps

- Workflow, mission, run, and artifact pages are still lighter than the profile surface.
- The shell teaches the nouns correctly, but only profiles currently expose a deeper builder/detail path.
